Session

interface Session(source)

Handle on a single logical conversation slice.

A session remembers every message that flows through it (both directions) so it can later wipe them from Telegram in bulk — optionally filtered by an arbitrary predicate.

Sessions never rely on CoroutineContext/thread-local state: callers pass the instance explicitly (via DI, with(session) { … }, or an extra send parameter), so tracking can never be lost when handlers launch child coroutines.
